Basic Curl Calls to api.github.com

The Github documentation for creating a new repository gives us the API call form: POST /user/repos and then defines a structure for the JSON string to be attached in the header. Many of the fields have default boolean values and the only strictly necessary field is name

An example JSON string for creating a new repo would be
“{“name”:”my new repository”,”description”:”this is my new github repo”}”


We will be using curl to make our API calls to the Github server. To attach this to the header of our HTTPS call we will use the -d/--data flag. Unlike calls that just request data from the Github server, calls on this API that create/modify/delete repositiories will require user credentials. We attach this to the call with the -u/--user flag, wich can be just the username if you would like the server to prompt for the password. Putting together this JSON string, the required user credentials (with your own credentials substituted for USERNAME:PASSWORD), and the provided URL a simple curl call to create a new repository would be as follows:

curl -u "USERNAME:PASSWORD" https://api.github.com/user/repos -d '{"name":"my new repository","description":"this is my new github repo"}'

This will suffice to create a new repository on the Github servers and will output the returned JSON string with information about the newly created repository. But this is hardly a time saver to type out API URLs and JSON strings by hand every time.
