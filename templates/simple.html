{% extends "main.html" %} {% block jumbo_title %} Passing Primitive Types {% endblock %} {% block body %} The simplest case is passing just a few primitive types such as ints or Strings to the activity you are launching. This is done using the optional Bundle that is included within the intent used to launch the activity. The user can create a Bundle directly and use the Intent.PutExtras() method or use the convince methods of the Intent class to add items to the Bundle easily.

As can be seen in the snippet below after creating an Intent that will launch the SnippetViewActivity the URL and JSON URL of the requested playlist are added to the intents Bundle using the PutExtra method. This differs from adding to the Bundle directly in that it is not necessary to call the corresponding method to the type being added such as putString() or putInt() as is needed when using the Bundle methods.

In the snippet below we see how to then retrieve the data that was packed into the Bundle before starting the activity. First the Bundle is retrieved from the Intent using the getExtras() method and then the data is extracted using the key strings. There is an inverse get*() method for every type of value that can be put*() in a Bundle.

Always remember to do null checks in Java, If no extra data was placed in the intent launching the activity then getExtras() will return null. It is also recommended to use Bundle.containsKey() before extracting a value to confirm that the key has been set.

This example was simple because the object that the user wanted to view was not yet retrieved from the Internet and it was only necessary to pass the URL strings to the next activity. In the next section we will cover how to pass more complicated data between activity.
